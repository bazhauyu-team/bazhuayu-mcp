# 八爪鱼 MCP 是什么？

> English: [Overview.en.md](Overview.en.md)

八爪鱼 MCP 可以让 ChatGPT、Claude、Cursor、VS Code GitHub Copilot、Codex CLI、Gemini CLI 等 AI 工具直接调用八爪鱼的采集能力。你可以用自然语言告诉 AI 要采集什么，AI 再通过 MCP 去搜索模板、创建云采集任务，并把导出的结构化数据返回给你。

它不是用来替代八爪鱼客户端、网页端或 OpenAPI 的。更准确地说，MCP 是给 AI 准备的一层入口：过去你需要自己打开八爪鱼、找模板、配参数、跑任务、导数据；现在这些步骤可以交给支持 MCP 的 AI 客户端来串起来。

## 什么是 MCP？

MCP，全称 Model Context Protocol，可以理解为 AI 工具连接外部服务的一套通用协议。

没有 MCP 时，AI 大多只能根据已有知识回答问题，或者根据你粘贴的内容做分析。有了 MCP，AI 可以在获得授权后调用外部工具，比如查数据、创建任务、读取结果、生成下载链接。

对用户来说，MCP 的意义很直接：如果一个服务支持 MCP，那么 ChatGPT、Claude、Cursor、Codex CLI 等客户端就可以用相对统一的方式接入它。你不需要为每个 AI 工具单独学习一套完全不同的对接方式。

## 八爪鱼 MCP 能做什么？

八爪鱼 MCP Server 把八爪鱼已有的模板搜索、云采集任务和数据导出能力，整理成 AI 更容易调用的工具。

推荐使用的主流程是：

```text
search_templates -> execute_task -> export_data
```

- `search_templates`：搜索采集模板，比如微博搜索、商品列表、搜索引擎结果等。
- `execute_task`：检查参数，创建并启动云采集任务。
- `export_data`：任务完成后导出数据，返回表格预览和下载链接。

这套设计的重点不是把所有底层 API 都暴露给 AI，而是把最常见的采集动作整理成少量清晰工具。这样 AI 更容易判断下一步该调用什么，也更不容易在复杂参数和细碎接口里迷失。

## 为什么已经有客户端、网页和 OpenAPI，还需要 MCP？

因为它们解决的是不同问题。

八爪鱼客户端和网页端更适合人来操作。比如你要设计一个复杂模板、调试页面元素、管理已有任务、查看采集结果，这些都更适合在可视化界面里完成。

OpenAPI 更适合开发者做系统对接。比如把八爪鱼接入企业内部系统、数据管道、定时任务或后台服务，开发者可以用代码精确控制请求、状态和异常处理。

MCP 则更适合 AI 参与的场景。你可以直接对 AI 说：

```text
帮我找一个微博搜索模板，采集 DeepSeek、Claude、OpenAI 这几个关键词，然后导出 CSV。
```

AI 可以通过 MCP 完成后面的动作：找模板、确认参数、创建任务、等待结果、导出数据。这种体验更接近“让 AI 帮我完成一件事”，而不是“我自己去调用一组 API”。

可以简单理解为：

| 方式 | 适合场景 | 核心价值 |
| --- | --- | --- |
| 客户端 / 网页端 | 人手动设计、调试和管理采集任务 | 可视化、可控、适合复杂配置 |
| OpenAPI | 程序员做系统集成 | 稳定、可编程、适合后台自动化 |
| MCP | AI 在对话中帮你执行采集流程 | 自然语言驱动、少配置、适合 AI Agent |

## 适用场景

八爪鱼 MCP 适合把网页数据采集交给 AI 来发起和串联，尤其适合已经有明确采集目标，但不想手动走完整操作流程的场景。

例如：

- 让 AI 搜索微博、新闻、论坛、内容平台上的关键词数据。
- 采集电商商品列表、价格、评论、店铺或类目信息。
- 整理门店、商家、地点、区域等本地生活和地图数据。
- 做市场调研、竞品分析、销售线索收集或舆情观察。
- 让 AI 在写报告、做分析或整理资料时，顺手补充结构化网页数据。
- 把同一套八爪鱼采集能力接入 ChatGPT、Claude、Cursor、VS Code、Codex CLI 等多个 AI 客户端。

如果你要从零设计一个复杂采集模板，八爪鱼客户端仍然是更合适的入口。如果你已经知道要采什么，或者希望 AI 帮你在已有模板中找到合适方案并执行任务，MCP 会更顺手。

## 能力边界

八爪鱼 MCP 调用的是八爪鱼的云采集能力。它能帮 AI 发起和管理采集流程，但不等于万能网页采集器。

它可以：

- 搜索八爪鱼采集模板。
- 根据模板要求填写参数并创建云任务。
- 在创建任务前做参数预检。
- 启动云采集任务并跟进结果。
- 导出结构化数据，返回预览表格和下载链接。
- 使用 OAuth 或 API Key 完成认证。

它不负责：

- 即时解析任意网页。
- 设计和调试复杂模板。
- 绕过网站登录、权限、风控、反爬或法律限制。
- 保证所有模板、所有网站、所有账号都一定采集成功。

任务能否顺利执行，仍然取决于模板是否支持云采集、账号权限和额度、目标网站变化、网络环境以及导出状态。免费版和基础版账号可以获得一定额度的 MCP 采集免费记录，适合先低成本试用；具体额度以产品当前规则为准。

## 谁适合使用？

八爪鱼 MCP 更适合这些人和团队：

- 想在 ChatGPT、Claude 等 AI 工具里直接发起采集任务的用户。
- 已经在使用八爪鱼模板和云采集，希望把能力接入 AI 客户端的用户。
- 需要把网页数据用于市场、运营、销售、研究、舆情或数据分析的团队。
- 希望让 AI Agent 自动补充实时网页数据的团队。
- 正在搭建内部 AI 助手、自动化流程或数据工具的开发者。

如果你想先跑通一次完整流程，可以阅读 [QuickStart](quickstart.md)，从搜索模板开始，完成一次任务创建和数据导出。
